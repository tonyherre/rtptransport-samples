WASM Congestion Control using RtpTransport

Sample running WASM-compiled GoogCC BWE on a live loopback PeerConnection via the experimental RtpTransport APIs.

RtpTransport support not found - check your browser version and flags as described below.

NOTE: This is a very early prototype. Actual web apps should perform congestion control on workers, once that's supported.

Requires a Chromium build >= 128.0.6564.0 started with the flags --force-fieldtrials=WebRTC-Bwe-InjectedCongestionController/Enabled --enable-blink-features=RTCRtpTransport

See googcc_runner.js for the interesting JS code making use of RtpTransport.
